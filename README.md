Программа с наименованием «Программа стеганографического коди-рования, декодирования и стегоанализа изображений» предназначена для стеганографического кодирования текстовых сообщений в изображения, стеганографического декодирования скрытых текстовых сообщений и стегоанализа, то есть обнаружения факта наличия скрытого сообщения в изображении. Программа корректно работает только с форматами, которые не используют сжатие с потерями (PNG, BMP, TIFF), максимальная длина сообщения ограничена емкостью выбранного метода. Для ее функционирования необходимы библиотеки PIL, NumPy, random, SciPy. При разработке была использована технология структурного программирования с элементами функционального подхода. 

После запуска работы программы пользователю открывается окно выбора режима: 1 – кодирование, 2 – декодирование, 3 – стегоанализ. В зависимости от введенного числа программа переходит к соответствующему режиму. 
Если выбран режим кодирования, то программа запрашивает номер ме-тода: 1 – LSB, 2 – RandomLSB, 3 – BlockLSB. Далее запрашивается путь к файлу исходного изображения, путь для сохранения результата. Для методов 2 и 3 вводится пароль. Далее программа пытается прочитать сообщение из файла messege.txt, если длина сообщения превышает емкость выбранного метода, то выводится сообщение об ошибке и работа режима прекращается. В противном случае вызывается соответствующая функция встраивания, после чего полученное сообщение сохраняется по указанному пути.
Если выбран режим декодирования, то программа запрашивает номер метода: 1 – LSB, 2 – RandomLSB, 3 – BlockLSB. Далее запрашивается путь к файлу изображения. Для методов 2 и 3 вводится пароль. После вызывается функция извлечения, которая восстанавливает бинарную строку, преобразует ее в текст и выводит на экран. Затем программа спрашивает, нужно ли со-хранить ли текст в файл, при положительном ответе запрашивается путь для сохранения.

Если выбран режима стегоанализа, то программа запрашивает номер атаки: 1 – визуальная, 2 – хи-квадрат, 3 – Regular-Singular. Далее запрашива-ется путь к файлу изображения. После вызывается функция атаки. Если выбрана визуальная атака, то создается и выводится изображение, отображаю-щее младшие биты, затем оно сохраняется по указанному пользователем пути. Для атаки хи-квадрат на экран выводится таблица с результатами значений хи-квадрат и p-value для каждой зоны и итоговый вердикт о наличии скрытого сообщения. Для Regular-Singular атаки выводится результат проверки гипотезы по каждому цветовому каналу для нескольких масок.
